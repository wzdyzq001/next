# AI 助手入口带入逻辑与聊天记录折叠优化 - Verification Checklist

## 入口带入逻辑验证

- [ ] Checkpoint 1: 从订单中心入口（无订单上下文）进入 AI 助手，不新增任何订单卡片消息，messages 数组长度与历史记录一致
- [ ] Checkpoint 2: 从订单详情页进入新订单（与上次进入的订单不同），消息列表末尾新增一条该订单的订单卡片
- [ ] Checkpoint 3: 从订单详情页重复进入同一订单，不新增订单卡片，消息列表与上次关闭时一致
- [ ] Checkpoint 4: 订单卡片展示的状态、价格、商品信息等与订单详情页实时一致

## 使用提醒卡片去重验证

- [ ] Checkpoint 5: 订单有活跃使用提醒 + 最新 3 条消息内已有同订单提醒卡片 → 进入时不新增提醒卡片
- [ ] Checkpoint 6: 订单有活跃使用提醒 + 最新 3 条消息内无同订单提醒卡片 → 进入时新增一条提醒卡片
- [ ] Checkpoint 7: 订单有活跃使用提醒 + 第 4 条及更早消息中有同订单提醒卡片 → 视为"不存在"，需新增
- [ ] Checkpoint 8: 新带入的提醒卡片样式、按钮交互与现有提醒卡片完全一致
- [ ] Checkpoint 9: 订单无活跃使用提醒时 → 进入时不触发提醒卡片去重逻辑

## 聊天记录折叠功能验证

- [ ] Checkpoint 10: 订单中心入口首次进入且历史消息 > 2 条 → 默认仅展示最新 2 条，更早的折叠
- [ ] Checkpoint 11: 订单中心入口且历史消息 ≤ 2 条 → 全部展示，无折叠区域
- [ ] Checkpoint 12: 折叠区域显示正确的被折叠消息条数（如"还有 6 条历史消息"）
- [ ] Checkpoint 13: 点击折叠区域后，所有历史消息展开显示，折叠区域消失

## 订单详情页折叠策略验证

- [ ] Checkpoint 14: 从订单详情页进入新订单 → 订单卡片之前的所有历史消息被折叠，折叠区域位于订单卡片上方（消息列表顶部）
- [ ] Checkpoint 15: 从订单详情页重复进入同一订单 → 保持上次的展开/折叠状态
- [ ] Checkpoint 16: 新订单进入时，折叠区域显示的条数为订单卡片之前的历史消息数

## 折叠状态持久化验证

- [ ] Checkpoint 17: 订单中心入口手动展开后，关闭再重新打开 → 保持展开状态
- [ ] Checkpoint 18: 订单中心入口手动折叠后，关闭再重新打开 → 保持折叠状态
- [ ] Checkpoint 19: 订单 A 详情页手动展开后，从订单 B 详情页进入 → 订单 B 为新订单，默认折叠
- [ ] Checkpoint 20: 订单 A 详情页手动展开后，再次从订单 A 详情页进入 → 保持展开状态
- [ ] Checkpoint 21: 刷新页面后，各入口的折叠状态保持不变（localStorage 持久化生效）

## 交互与视觉验证

- [ ] Checkpoint 22: 折叠/展开切换有平滑过渡动画，无明显卡顿
- [ ] Checkpoint 23: 折叠区域样式符合 Notion 风格，布局紧凑，与整体视觉协调
- [ ] Checkpoint 24: 移动端响应式布局下折叠区域展示正常，文字不溢出
- [ ] Checkpoint 25: 折叠状态下滚动消息列表行为正常，不出现滚动异常

## 回归测试验证

- [ ] Checkpoint 26: 消息发送功能正常，发送消息后自动滚动到底部
- [ ] Checkpoint 27: 订单卡片的所有按钮交互（查看券码、帮我约、使用提醒等）正常
- [ ] Checkpoint 28: 预约面板、提醒浮层、券码浮层的拉起与关闭正常
- [ ] Checkpoint 29: 订单选择器功能正常，选择订单后发送订单卡片
- [ ] Checkpoint 30: 重置会话功能正常，清空消息记录
- [ ] Checkpoint 31: 关闭 AI 助手后重新打开，消息记录持久化正常
