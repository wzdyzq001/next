# AI 助手自然语言交互 Demo - 验证清单

## 意图识别
- [ ] 输入"帮我约一下"，能正确识别为预约意图
- [ ] 输入"设置一个提醒"，能正确识别为提醒意图
- [ ] 输入"我的取餐码在哪"，能正确识别为取餐码查询意图
- [ ] 输入"外卖送到哪了"，能正确识别为配送查询意图
- [ ] 模糊输入（如包含多个关键词）能正确识别主意图

## 帮我约功能
- [ ] 需要预约的订单卡片展示时，显示预约引导提示语
- [ ] 至少有 3 种不同的预约提示模板（客流量大、临近假期、周末高峰等）
- [ ] 说"我想预约"后，AI 回复引导语并显示"帮我约"快捷按钮
- [ ] 只提供日期时，AI 追问时间和人数
- [ ] 只提供时间时，AI 追问日期和人数
- [ ] 只提供人数时，AI 追问日期和时间
- [ ] 提供日期+时间缺人数时，AI 追问人数
- [ ] 信息完整后，自动生成预约卡片（状态：预约确认中）
- [ ] 选择已过期/不可约时间，提示不可约并展示可约选项
- [ ] 点击"帮我约"快捷按钮，能拉起预约面板
- [ ] 预约卡片可以正常取消/修改

## 订单使用提醒
- [ ] 临期订单（≤7天）卡片旁显示气泡提醒
- [ ] 气泡在订单卡片展示 1.5 秒后弹出
- [ ] 气泡初始显示长文案："订单还有 X 天过期，请尽快使用～"
- [ ] 5 秒后气泡自动收起为短文案："临期提醒"
- [ ] 点击短文案可重新展开长文案
- [ ] 非临期订单不显示气泡提醒
- [ ] 用户进入 AI 助手时，临期订单消息额外增加提醒引导文案
- [ ] 说"帮我设置提醒"后，显示快捷日期选项
- [ ] 快捷选项包含：明天、后天、本周五/六/日、下周五/六/日、最后一天、过期前1天、过期前3天
- [ ] 日期重叠时优先展示近日期表达（明天/后天 > 本周X > 下周X）
- [ ] 所有可选日期不超过订单有效期，超出的选项自动隐藏
- [ ] 点击快捷选项，自动设置对应日期的提醒
- [ ] 手动输入日期也能正确解析
- [ ] 选择超过有效期的日期，提示"提醒日期不可以超过订单有效期"
- [ ] 超期提示附带"是否需要在过期前一天提醒？"的是/否选项
- [ ] 选择"是"，自动设置过期前一天提醒
- [ ] 选择"否"，继续让用户输入提醒日期
- [ ] 提醒设置成功后，生成使用提醒卡片
- [ ] 提醒卡片显示正确的提醒日期和订单信息

## 取餐码查询
### 无前置订单卡片场景
- [ ] 输入"取餐码"且无前置订单时，自动查询待取餐订单
- [ ] 找到 1 个待取餐订单：发送订单卡片（含取餐码）
- [ ] 找到多个待取餐订单：展示多个简化订单卡片
- [ ] 未找到待取餐订单：提示 + "选择订单"按钮
- [ ] 点击"选择订单"按钮，拉起订单选择浮层
- [ ] 从订单选择浮层选择订单后，自动查询取餐码

### 有前置订单卡片 - 非餐饮
- [ ] 非餐饮订单查询取餐码，提示"不是餐饮订单，没有取餐码"
- [ ] 提示后附带"选择订单"按钮
- [ ] 点击"选择订单"按钮，拉起订单选择浮层

### 有前置订单卡片 - 餐饮待使用
- [ ] 仅券码核销的订单：提示"还未使用没有取餐码，可出示券码验券"
- [ ] 仅券码核销的订单：附带"查看券码"按钮
- [ ] 点击"查看券码"按钮，拉起券码浮层
- [ ] 支持点单核销的订单：提示"还未使用没有取餐码"
- [ ] 支持点单核销的订单：附带"是否需要我帮你点单？"是/否选项
- [ ] 选择"是"，触发点单流程，订单状态变为"已确认/备餐中"
- [ ] 点单完成后自动展示取餐码
- [ ] 仅配送的订单：提示"还未使用没有取餐码，可以配送到家"
- [ ] 仅配送的订单：附带"是否需要我帮你预约配送？"是/否选项
- [ ] 选择"是"，触发配送预约流程，模拟配送状态流转

### 有前置订单卡片 - 餐饮已完成
- [ ] 券码核销（抖音端内）的已完成订单：提示"券码核销无取餐码" + "选择订单"按钮
- [ ] 券码核销（端外）的已完成订单：提示"没有找到取餐码，如果在商家App/微信小程序核销，可通过原渠道查找取餐码"
- [ ] 点单核销的已完成订单：展示取餐码 + 取餐进度信息
- [ ] 配送核销的已完成订单：提示"配送订单无取餐码" + 展示配送进度

## 配送进度查询
- [ ] 输入"配送"、"物流"等关键词，识别为配送查询意图
- [ ] 有前置配送中订单：展示配送进度条、骑手信息、预计送达时间
- [ ] 无前置订单时，自动查询配送中订单并展示
- [ ] 非配送订单查询配送，引导选择订单
- [ ] 配送进度展示与订单卡片中的扩展信息一致

## 通用交互
- [ ] 所有快捷按钮点击后自动发送对应消息
- [ ] AI 回复有适当延迟（300-800ms），模拟真实思考
- [ ] 多轮对话中状态保持，不会意外中断
- [ ] 状态变更时有视觉反馈（卡片更新、消息动画等）
- [ ] 所有流程都有明确的出口，不会死循环
- [ ] 刷新页面后数据状态保持（localStorage 持久化）
