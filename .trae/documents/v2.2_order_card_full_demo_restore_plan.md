# AI 助手订单卡片完整还原 DEMO 开发计划

## 一、背景与目标

当前 AI 助手聊天流中的订单卡片（FullOrderCard）虽然已经实现了基础功能，但与 DEMO 模式下的订单卡片相比，在以下方面存在差距：

1. **数据层**：`convertOrderListItemToCardData` 函数的实现没有完全按照 DEMO 的逻辑，导致部分行业、商品类型、履约方式、订单状态组合下的标签、按钮、扩展层展示不正确
2. **交互层**：按钮点击交互（使用提醒、帮我约、查看券码）没有实现为拉起浮层弹窗的方式，而是使用了功能卡片或简单回调
3. **弹窗层**：券码弹窗（VoucherCodeSheet）未在 AI 助手聊天流中集成

**目标**：严格按照 DEMO 示例中的各行业、商品类型、履约方式和订单状态的展示逻辑进行完整还原开发，确保每个交互场景都与 DEMO 保持完全一致。

## 四、详细实现方案

### Step 1：重写数据适配器 - 完整映射 DEMO 逻辑

**文件**：`src/components/AiAssistant/orderDataAdapter.ts`

**目标**：将 `convertOrderListItemToCardData` 函数完全按照 DEMO 的逻辑重写，确保：

1. **标签生成**：根据行业、商品类型、履约方式、退款规则、有效期等生成与 DEMO 一致的标签
2. **按钮生成**：根据行业、商品类型、订单状态、履约方式生成与 DEMO 完全一致的按钮组合
3. **扩展层生成**：根据订单状态生成对应的扩展层（progress、hotel_stay、refund、payment_countdown 等）
4. **状态映射**：OrderListItem 的 statusText → OrderCardData 的 orderStatus/orderStatusLabel 精确映射
5. **分类映射**：OrderListItem 的 category → OrderCardData 的 category/categoryLabel 精确映射
6. **商品类型映射**：OrderListItem 的 hotelProductType/scenicProductType → OrderCardData 的 productType/productTypeLabel
7. **券码信息**：如果有 voucherCode 字段，生成 voucherInfo

**关键实现点**：
- 创建状态映射表：`statusText → { orderStatus, orderStatusLabel, statusColor }`
- 创建分类映射表：`OrderListItem.category → { category, categoryLabel }`
- 创建商品类型映射表：`hotelProductType/scenicProductType → { productType, productTypeLabel }`
- 创建履约方式映射表：`fulfillmentModes → redeemTypes`
- 创建标签生成规则：按行业×商品类型×履约方式×退款规则×有效期生成
- 创建按钮生成规则：按行业×商品类型×订单状态×履约方式生成
- 创建扩展层生成规则：按订单状态×行业生成对应的扩展层数据

### Step 2：完善 AiAssistantProvider - 添加券码弹窗状态管理

**文件**：`src/components/AiAssistant/AiAssistantProvider.tsx`

**目标**：添加券码弹窗的状态管理和方法

**新增状态**：
- `voucherSheetOpen: boolean`
- `voucherSheetData: { code, number, validDate, notes } | null`

**新增方法**：
- `openVoucherSheet(voucherData)` - 打开券码弹窗
- `closeVoucherSheet()` - 关闭券码弹窗

**修改 executeAction**：
- 当 action.label 包含"使用提醒"时，调用 `openReminderSheet` 直接拉起提醒弹窗（而不是生成功能卡片）
- 当 action.label 包含"帮我约"时，调用 `openReservationPanel` 直接拉起预约弹窗（而不是生成功能卡片）
- 当 action.label 包含"查看券码"时，调用 `openVoucherSheet` 直接拉起券码弹窗（而不是生成功能卡片）

### Step 3：完善 AiAssistantOverlay - 集成券码弹窗

## 五、风险与注意事项

### 5.1 数据完整性风险
- OrderListItem 中的字段可能不足以完全还原 DEMO 的所有展示
- 对于缺失的数据，使用合理的默认值或降级方案

### 5.2 兼容性风险
- 修改数据适配器可能影响已有的订单列表页、详情页功能
- 确保现有功能不被破坏

### 5.3 弹窗交互一致性
- 确保三种弹窗（提醒、预约、券码）的交互方式与 DEMO 完全一致
- 弹窗位置、动画、关闭方式都需要对齐

### 5.4 分类映射准确性
- OrderListItem 的 category 枚举与 OrderCardData 的 category 枚举不完全一致
- 需要建立精确的映射关系
